# AI Tools Comparison Guide

이 문서는 수업 초반에 직접 자주 헷갈리는 AI 도구의 역할을 구분하기 위한 참고 자료입니다.

## 왜 도구를 구분해야 할까요?

AI 도구는 모두 비슷해 보이지만 실제 용도는 조금씩 다릅니다. 코드를 만들 때 좋은 도구, 문서를 요약할 때 좋은 도구, 최신 정보를 검색할 때 좋은 도구가 다를 수 있습니다.

 “어떤 도구가 제일 좋은가”보다 “내가 지금 하려는 일에 어떤 도구가 맞는가”를 먼저 판단해야 합니다.

## 대표 도구 비교

| 도구 | 주된 사용 목적 | 수업에서의 사용 방식 |
| --- | --- | --- |
| Codex | 코드 작성, 수정, 리뷰, 디버깅 | Python, FastAPI, Supabase 예제를 만들고 오류를 함께 해결합니다. |
| ChatGPT | 개념 설명, 코드 초안, 문서 작성 | 어려운 개념을 쉽게 설명받거나 README 초안을 작성할 때 사용합니다. |
| Claude | 긴 글 분석, 요구사항 정리, 코드 리뷰 설명 | 긴 README나 프로젝트 요구사항을 정리할 때 비교 도구로 활용할 수 있습니다. |
| Gemini | Gemini SDK 기반 LLM 실습, REST 호출 구조 이해 보충 | 무료 범위가 있는 LLM 실습 예시로 사용합니다. 기본 예제는 SDK 방식으로 진행하고, REST 방식은 요청 URL, 헤더, JSON payload 구조를 이해하는 보충 자료로 확인합니다. 실제 조건은 사용 시점의 공식 화면에서 확인합니다. |
| NotebookLM | 내가 넣은 문서 기반 요약과 질의응답 | 강의 자료나 프로젝트 문서를 넣고 요약, 핵심 질문 만들기에 사용합니다. |
| Perplexity AI | 웹 검색 기반 답변과 출처 확인 | 공식 문서 위치나 최신 기술 변경 사항을 확인할 때 사용합니다. |

## 수업 중 추천 사용 흐름

```text
1. 개념이 어렵다
 -> ChatGPT, Claude, Gemini에게 쉬운 설명을 요청합니다.

2. 코드가 필요하다
 -> Codex에게 요구사항, 파일 위치, 기대 동작을 함께 알려줍니다.

3. 최신 정보가 필요하다
 -> Perplexity AI 또는 공식 문서를 통해 출처를 확인합니다.

4. 수업 자료를 요약하고 싶다
 -> NotebookLM에 문서를 넣고 요약과 질문 생성을 요청합니다.

5. 최종 코드가 맞는지 확인하고 싶다
 -> Codex에게 코드 리뷰와 실행 검증 체크리스트를 요청합니다.
```

## 실습 질문 예시

```text
Codex에게:
이 FastAPI 코드에서 초보자가 실수하기 쉬운 부분을 주석으로 설명해 주세요.

NotebookLM에게:
이 강의 README를 기반으로 오늘 수업 핵심 질문 5개를 만들어 주세요.

Perplexity AI에게:
FastAPI의 Response Model 공식 문서 위치를 알려주고, 핵심 내용을 요약해 주세요.
```

## 주의할 점

- AI 답변은 항상 검증합니다.
- API key나 비밀번호를 AI 채팅창에 그대로 입력하지 않습니다.
- 최신 요금, 모델명, 제한 사항은 공식 문서 또는 서비스 화면에서 다시 확인합니다.
- 수업 과제 제출물에는 “AI가 만든 답변”이 아니라 “내가 이해하고 검증한 내용”을 적습니다.
