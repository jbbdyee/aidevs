# Prompt And Answer Validation Guide

이 문서는 좋은 질문을 작성하고 AI 답변을 검증하는 방법을 정리한 자료입니다.

## 좋은 프롬프트의 기본 구조

```text
역할(Role):
너는 Python 백엔드 입문 멘토야.

맥락(Context):
 변수, 함수, 클래스까지 배웠고 FastAPI는 처음이야.

요청(Instruction):
아래 코드를 실행 순서대로 설명해줘.

출력 형식(Output):
표로 정리하고, 마지막에 실습 체크리스트를 추가해줘.

검증 기준(Check):
실행 명령과 예상 결과도 함께 알려줘.
```

## 프롬프트 작성 예시

### 코드 설명 요청

```text
너는 초보자를 돕는 Python 멘토입니다.
아래 FastAPI 코드를 한 줄씩 설명해 주세요.
각 줄이 왜 필요한지, 실행하면 어떤 URL에서 확인할 수 있는지도 알려 주세요.
```

### 오류 해결 요청

```text
아래 오류 메시지를 보고 원인을 분석해 주세요.
1. 오류가 발생한 가능성이 높은 위치
2. 확인해야 할 명령
3. 수정 방법
4. 다시 실행할 명령
순서로 설명해 주세요.
```

### 코드 리뷰 요청

```text
아래 코드를 리뷰해 주세요.
리뷰 관점은 FastAPI 구조, Pydantic 검증, 보안, 초보자 가독성입니다.
문제가 있으면 파일명과 수정 방향을 함께 제안해 주세요.
```

## AI 답변 검증 순서

```text
1. 답변이 내 질문에 실제로 답했는지 확인합니다.
2. 코드가 현재 폴더 구조와 맞는지 확인합니다.
3. 실행 명령을 직접 실행합니다.
4. 오류가 나면 오류 메시지를 다시 AI에게 전달합니다.
5. API key, 비밀번호, 토큰이 노출되지 않았는지 확인합니다.
6. 비용이 발생하는 API 호출인지 확인합니다.
7. 최신 정보가 필요한 내용은 공식 문서로 다시 확인합니다.
```

## 초보자가 자주 하는 실수

| 실수 | 왜 문제인가요? | 개선 방법 |
| --- | --- | --- |
| “다 만들어줘”라고만 질문함 | 요구사항이 불명확해 결과가 흔들립니다. | 기능, 입력, 출력, 파일 위치를 함께 적습니다. |
| 실행하지 않고 복사만 함 | 코드가 현재 환경에서 동작하는지 알 수 없습니다. | 반드시 터미널에서 실행합니다. |
| API key를 코드에 직접 적음 | GitHub에 올라가면 보안 사고가 됩니다. | `.env` 파일과 환경 변수를 사용합니다. |
| 최신 정보를 AI 답변만 믿음 | 모델이 오래된 정보를 말할 수 있습니다. | 공식 문서와 서비스 화면을 확인합니다. |

## 수업 중 검증 질문

```text
이 답변에서 내가 직접 확인해야 할 부분은 무엇인가요?
이 코드가 실행되는지 확인하는 명령을 알려 주세요.
보안상 위험한 부분이 있는지 점검해 주세요.
초보자 입장에서 이해하기 어려운 줄에 주석을 추가해 주세요.
```
