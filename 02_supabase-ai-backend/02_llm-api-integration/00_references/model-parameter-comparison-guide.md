# Model Parameter Comparison Guide

LLM API를 사용할 때는 모델 이름만 고르는 것이 아니라, provider와 파라미터, 비용 기준을 함께 생각해야 합니다.

이 과정에서는 다음 기준으로 정리합니다.

```text
02~04 기본 provider: Gemini
기본 Gemini 모델 예시: gemini-2.5-flash-lite
OpenAI 선택 모델 예시: gpt-4.1-mini
기본 실습 방식: mock-first
```

## provider 선택 기준

| 구분 | 이 과정에서의 역할 |
| --- | --- |
| Gemini | 02~04 과정의 기본 LLM 실습 provider입니다. 무료 범위를 활용해 기본 호출 흐름을 익히는 데 사용합니다. |
| OpenAI | 선택 비교용 provider입니다. 기존 예제 파일은 유지하며, 필요할 때 모델 응답 차이나 API 사용 방식을 비교합니다. |
| Mock | 비용 없이 요청/응답 구조를 이해하기 위한 기본 실습 방식입니다. |

## 모델 선택 기준

| 기준 | 확인할 내용 |
| --- | --- |
| 비용 | 입력 token과 출력 token에 따라 비용이 달라질 수 있습니다. |
| 응답 속도 | 실시간 챗봇에서는 빠른 응답이 중요합니다. |
| 응답 품질 | 설명, 요약, 코드 생성 등 작업에 따라 품질 차이가 날 수 있습니다. |
| context 길이 | 긴 문서나 긴 대화를 얼마나 처리할 수 있는지 확인합니다. |
| API 안정성 | 학습과 과제에서는 실행이 단순하고 재현 가능한 구조가 중요합니다. |

## 주요 파라미터 비교

| 파라미터 | 의미 | 기본 학습 기준 |
| --- | --- | --- |
| `temperature` | 응답의 다양성과 창의성을 조절합니다. | 코드 설명은 낮게, 아이디어 생성은 조금 높게 |
| `top_p` | 다음 단어 후보 범위를 조절합니다. | 초반에는 기본값에 가깝게 유지 |
| `max_tokens` | 출력 길이를 제한합니다. | 비용과 응답 길이를 고려해 필요한 만큼 설정 |
| provider별 고급 설정 | 특정 모델이나 SDK에서만 제공될 수 있습니다. | 공식 문서 확인 후 선택 적용 |

## mock과 실제 호출 비교

| 구분 | mock 호출 | 실제 API 호출 |
| --- | --- | --- |
| 비용 | 없음 | 발생 가능 |
| API key | 필요 없음 | 필요 |
| 네트워크 | 필요 없음 | 필요 |
| 응답 품질 | 직접 작성한 예시 응답 | 실제 모델 응답 |
| 학습 용도 | 구조 이해, 테스트, 과제 기본값 | 선택 실습, 실제 서비스 연결 |

## 권장 진행 방식

```text
1. mock으로 요청/응답 구조를 먼저 완성합니다.
2. provider, model, actual_api_called 필드를 응답에 포함합니다.
3. API key와 비용 조건을 확인합니다.
4. Gemini 실제 호출 예제를 실행합니다.
5. 필요하면 OpenAI 예제와 비교합니다.
```

## 응답 예시

```json
{
  "provider": "gemini",
  "model": "gemini-2.5-flash-lite",
  "actual_api_called": false,
  "answer": "mock 응답입니다."
}
```

`actual_api_called`가 `false`이면 실제 API를 호출하지 않은 상태입니다. 과제와 실습에서는 이 값을 명확하게 표시해야 합니다.
