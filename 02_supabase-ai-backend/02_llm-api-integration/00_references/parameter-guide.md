# Parameter Guide

LLM API 파라미터는 모델이 답변을 만드는 방식을 조절하는 설정값입니다.

처음에는 모든 파라미터를 외우기보다, 아래 세 가지를 먼저 이해하면 충분합니다.

## temperature

`temperature`는 응답의 다양성과 예측 가능성을 조절합니다.

```text
낮은 값: 더 안정적이고 반복 가능한 답변
높은 값: 더 다양하고 창의적인 답변
```

추천 기준:

| 상황 | 예시 값 |
| --- | --- |
| 코드 설명, 오류 분석, API 문서 작성 | `0.2` ~ `0.4` |
| 아이디어 생성, 문구 제안 | `0.6` ~ `0.8` |

## top_p

`top_p`는 모델이 다음 단어를 고를 때 후보 범위를 얼마나 넓게 볼지 조절합니다.

초보자 단계에서는 `top_p`를 자주 바꾸기보다 기본값에 가깝게 두고, 먼저 `temperature`의 차이를 이해하는 것이 좋습니다.

## max_tokens

`max_tokens`는 모델이 생성할 수 있는 최대 출력 길이를 제한합니다.

출력이 길어질수록 비용이 늘어날 수 있으므로, 실제 API 호출 전에는 필요한 만큼만 설정합니다.

예시:

```text
짧은 요약: 200~400
중간 길이 설명: 500~1000
긴 문서 초안: 1000 이상
```

## thinkingLevel 같은 고급 설정

일부 모델이나 SDK는 추론 강도를 조절하는 고급 설정을 제공할 수 있습니다. 하지만 모든 provider와 모든 모델이 같은 이름의 파라미터를 제공하는 것은 아닙니다.

이 과정에서는 고급 설정을 외우기보다 다음 기준을 기억합니다.

```text
1. 기본 파라미터로 먼저 구조를 이해합니다.
2. provider별 고급 설정은 공식 문서에서 현재 지원 여부를 확인합니다.
3. 비용이 달라질 수 있는 설정은 실제 호출 전에 반드시 확인합니다.
```
