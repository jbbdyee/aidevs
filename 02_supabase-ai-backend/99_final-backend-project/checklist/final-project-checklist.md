# 최종 프로젝트 체크리스트

이 문서는 최종 백엔드 프로젝트를 제출하기 전에 수강생이 스스로 확인하는 체크리스트입니다.
점수를 매기기 위한 문서라기보다, 프로젝트가 처음 보는 사람에게도 실행 가능하고 설명 가능한 상태인지 점검하는 기준입니다.

## 필수 점검 항목

| 점검 항목 | 필수 확인 기준 |
| --- | --- |
| 실행 가능성 | FastAPI 서버가 실행되고 `GET /health`가 동작합니다. |
| API 설계 | 최소 3개 이상의 endpoint가 있고 HTTP Method가 의미에 맞습니다. |
| Pydantic | 요청 Body 검증 모델이 있습니다. |
| LLM 흐름 | mock 또는 Gemini 호출 흐름이 명확합니다. |
| Supabase | 최소 1개 테이블과 연결되어 저장 또는 조회가 됩니다. |
| 문서화 | API 설계, DB 설계, Swagger 테스트 결과가 정리되어 있습니다. |
| 보안 | `.env`를 제출하지 않고 민감 정보를 문서에 노출하지 않습니다. |
| 오류 대응 | 오류가 발생했을 때 원인과 해결 기록을 남깁니다. |

## 선택 점검 항목

| 항목 | 기준 |
| --- | --- |
| Gemini 실제 호출 | API key와 비용 제한을 고려해 실제 호출을 연결했습니다. |
| Redis | TTL 기반 캐시를 간단히 적용했습니다. |
| Auth/JWT/RLS | 사용자별 접근 제어를 설명하거나 구현했습니다. |
| 구조 분리 | 코드가 길어졌을 때 파일 역할을 나누었습니다. |
| 테스트 | pytest 또는 TestClient로 핵심 API를 검증했습니다. |
