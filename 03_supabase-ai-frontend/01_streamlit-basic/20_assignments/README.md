# 20_assignments

수업 후 제출 과제 안내입니다.

과제는 예제 코드를 그대로 제출하는 것이 아니라, 배운 Streamlit 기본 문법을 사용해 자신만의 작은 화면을 구성하는 방식으로 진행합니다. 복잡한 기능보다 “실행되는 화면”, “명확한 입력”, “이해 가능한 출력”이 중요합니다.

## 제출 과제

```text
assignment-01-self-introduction-app.md
assignment-02-basic-dashboard-app.md
```

## 제출 기준

- Streamlit 앱이 `streamlit run`으로 실행되어야 합니다.
- 입력값을 최소 2개 이상 사용해야 합니다.
- 결과 출력 영역이 입력 영역과 구분되어야 합니다.
- 코드에 본인이 이해한 주석을 추가해야 합니다.
- 실행 화면을 캡처하거나 실행 결과를 문서에 정리해야 합니다.

## 권장 진행 순서

```text
1. 예제 파일 중 가장 비슷한 파일을 하나 고릅니다.
2. 파일을 새 이름으로 복사하거나 새 파일을 만듭니다.
3. 제목, 입력 항목, 출력 문구를 과제 요구사항에 맞게 바꿉니다.
4. streamlit run으로 실행해 화면을 확인합니다.
5. 오류가 있으면 터미널 메시지의 파일명과 줄 번호를 확인합니다.
6. 완성 후 실행 방법과 화면 설명을 과제 문서에 정리합니다.
```

## 평가 기준

| 항목 | 확인 기준 |
| --- | --- |
| 실행 가능성 | `streamlit run` 명령으로 오류 없이 실행됩니다. |
| 입력 처리 | 텍스트, 숫자, 선택 입력 중 2개 이상을 사용합니다. |
| 화면 구성 | 입력 영역과 결과 영역이 구분되어 있습니다. |
| 코드 이해 | 주요 코드에 직접 이해한 주석이 들어 있습니다. |
| 결과 정리 | 실행 방법과 화면 동작 설명이 문서에 정리되어 있습니다. |
