# Streamlit UI Components Notes

## 핵심 UI 컴포넌트

Streamlit 컴포넌트는 사용자의 행동을 Python 변수로 바꿔 주는 도구입니다. 사용자가 클릭하거나 입력하거나 선택하면, 그 값이 코드 안에서 조건문, 필터, 출력 문구에 사용됩니다.

- `st.button()`: 버튼 클릭 이벤트를 처리합니다.
- `st.checkbox()`: 참/거짓 선택을 받습니다.
- `st.slider()`: 숫자 범위 입력을 받습니다.
- `st.form()`: 여러 입력값을 한 번에 제출합니다.
- `st.form_submit_button()`: 폼 안의 입력값을 제출하는 버튼입니다.
- `st.dataframe()`: 스크롤 가능한 표를 표시합니다.
- `st.table()`: 정적인 표를 표시합니다.
- `st.line_chart()`: 선 그래프를 표시합니다.
- `st.bar_chart()`: 막대 그래프를 표시합니다.
- `st.file_uploader()`: 파일 업로드 입력을 받습니다.
- `st.metric()`: 핵심 숫자나 상태값을 카드처럼 표시합니다.

## 버튼과 폼의 차이

버튼은 한 가지 행동을 바로 실행할 때 사용합니다.

```text
버튼 클릭
-> 바로 결과 표시
```

폼은 여러 입력값을 먼저 작성한 뒤 한 번에 제출할 때 사용합니다.

```text
이름 입력
관심 분야 선택
의견 작성
-> 제출 버튼 클릭
-> 결과 표시
```

사용자 질문, 검색 조건, 필터 조건처럼 여러 값을 모아 처리해야 할 때는 폼이 더 적합합니다.

## pandas를 사용하는 이유

Streamlit에서 표와 차트를 다룰 때 데이터는 보통 pandas DataFrame 형태로 사용합니다.

이 단원에서는 다음 정도만 익히면 충분합니다.

- `pd.DataFrame()`으로 표 데이터 만들기
- 컬럼 이름 확인하기
- 조건으로 행 필터링하기
- 합계와 평균 계산하기
- CSV 파일을 `pd.read_csv()`로 읽기

## `st.dataframe`과 `st.table` 차이

```text
st.dataframe
-> 스크롤과 정렬 등 인터랙션이 있는 표를 보여 줄 때 사용합니다.

st.table
-> 작은 데이터를 정적인 표로 보여 줄 때 사용합니다.
```

대화 이력이나 서비스 로그처럼 행이 많아질 수 있는 데이터는 `st.dataframe`이 더 적합합니다.

## 파일 업로드 흐름

CSV 업로드 예제는 이후 로그 파일, 피드백 파일, 테스트 결과 파일을 화면에서 확인하는 기초가 됩니다.

```text
1. st.file_uploader로 파일을 받습니다.
2. 파일이 업로드되었는지 확인합니다.
3. pd.read_csv로 DataFrame을 만듭니다.
4. st.dataframe으로 표를 확인합니다.
5. 필요한 컬럼을 선택해 차트를 만듭니다.
```

## 화면 구성 기준

대시보드 화면은 아래처럼 나누면 읽기 쉽습니다.

```text
sidebar
-> 필터와 설정 입력

상단 metric
-> 가장 중요한 지표

본문 표
-> 원본 데이터 확인

차트
-> 변화와 비교 확인
```

## 학습 원칙

pandas 문법을 많이 외우는 것보다, Streamlit 화면을 만들기 위해 필요한 데이터 모양을 이해하는 데 집중합니다.

- 데이터가 리스트인지 딕셔너리인지 확인합니다.
- DataFrame의 컬럼 이름을 확인합니다.
- 필터 결과가 화면에 잘 반영되는지 확인합니다.
- 표와 차트가 같은 데이터를 기준으로 연결되어 있는지 확인합니다.
