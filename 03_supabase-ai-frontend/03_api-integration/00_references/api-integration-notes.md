# API Integration Notes

## 기본 흐름

```text
사용자 입력
-> Streamlit 버튼 클릭
-> httpx로 API 요청
-> FastAPI에서 JSON 응답 반환
-> Streamlit 화면에 결과 출력
```

## 주요 개념

- `GET`: 서버에서 데이터를 조회할 때 사용합니다.
- `POST`: 서버에 데이터를 보낼 때 사용합니다.
- `JSON`: 프론트엔드와 백엔드가 주고받는 대표적인 데이터 형식입니다.
- `status code`: 요청 성공 또는 실패 상태를 나타내는 숫자입니다.
- `timeout`: 서버 응답을 기다릴 최대 시간입니다.
- `payload`: POST 요청으로 백엔드에 보내는 JSON 데이터입니다.

## GET 요청과 POST 요청

GET 요청은 보통 서버에서 값을 가져올 때 사용합니다.

```text
GET /health
GET /api/courses
```

POST 요청은 보통 사용자가 입력한 값을 서버로 보낼 때 사용합니다.

```text
POST /api/message
POST /api/score-feedback
```

## API 주소 관리

프론트엔드 코드에서 API 주소를 여러 곳에 직접 쓰면 나중에 수정하기 어렵습니다. 그래서 보통 기본 주소를 변수로 분리합니다.

```python
API_BASE_URL = "http://127.0.0.1:8000"
```

실제 백엔드 주소가 바뀌면 이 값만 바꾸면 됩니다.

## 자주 확인할 것

- 백엔드 서버가 실행 중인가?
- API 주소가 정확한가?
- 요청 method가 GET인지 POST인지 맞는가?
- JSON key 이름이 백엔드 모델과 일치하는가?
- 응답 status code가 200인지 확인했는가?
- 응답 JSON에 화면에서 사용할 key가 있는가?

## 오류 처리 기준

프론트엔드에서는 오류를 그대로 터미널에만 두지 않고, 사용자가 이해할 수 있는 문장으로 화면에 표시해야 합니다.

```text
서버가 꺼져 있음
-> "백엔드 서버에 연결할 수 없습니다."

응답이 너무 오래 걸림
-> "요청 시간이 초과되었습니다."

응답 key가 없음
-> "응답 형식이 예상과 다릅니다."
```

이 기준은 이후 챗봇 UI와 서비스 로그 화면에서도 그대로 사용됩니다.
