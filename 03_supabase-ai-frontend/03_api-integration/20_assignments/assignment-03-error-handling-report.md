# Assignment 03: Error Handling Report

## 목표

API 연동 중 발생할 수 있는 오류 상황을 직접 확인하고 기록합니다.

이 과제의 목적은 오류를 피하는 것이 아니라, 오류가 발생했을 때 원인을 확인하고 사용자에게 어떤 메시지를 보여 줄지 판단하는 것입니다.

## 확인할 오류 상황

- FastAPI 서버를 끄고 Streamlit에서 API 호출하기
- 잘못된 API 경로 호출하기
- POST 요청에서 필수 입력값 비워 보기
- 응답 JSON key 이름이 예상과 다를 때 처리하기
- timeout 시간을 아주 짧게 설정해 보기

## 제출 내용

- 실행한 명령어
- 발생한 오류 메시지
- 원인 분석
- 수정 방법
- 최종 확인 결과

## 보고서 작성 형식

```text
오류 상황:

실행 명령:

화면에 표시된 메시지:

터미널 오류 메시지:

원인:

수정 방법:

다시 실행한 결과:
```

## 확인 기준

- 정상 상황과 오류 상황을 모두 기록했습니다.
- 사용자가 이해할 수 있는 오류 안내 문장을 제안했습니다.
- 단순히 “안 됨”이 아니라 status code, timeout, 연결 실패 중 무엇인지 구분했습니다.
