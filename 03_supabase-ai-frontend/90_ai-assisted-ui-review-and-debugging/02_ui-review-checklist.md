# 02. UI Review Checklist

최종 제출 전 아래 항목을 빠르게 점검합니다.

## 화면 흐름

- 첫 화면에서 사용자가 무엇을 해야 하는지 자연스럽게 보이는가?
- 입력 영역과 결과 영역이 구분되어 있는가?
- 버튼을 눌렀을 때 화면 반응이 있는가?
- 오류가 발생했을 때 조용히 실패하지 않고 메시지가 보이는가?

## API 연결

- `API_BASE_URL`을 코드에 흩뿌리지 않고 한 곳에서 읽는가?
- 백엔드가 꺼졌을 때 사용자에게 확인 메시지를 보여 주는가?
- timeout을 설정했는가?
- status code가 실패일 때 오류를 처리하는가?

## 상태 관리

- `st.session_state` key를 사용하기 전에 초기화했는가?
- 로그인 token을 화면에 그대로 출력하지 않는가?
- 로그아웃 시 token과 사용자 상태를 정리하는가?
- 대화 이력이 새로고침 흐름에서도 예상대로 동작하는가?

## 보안

- 프론트엔드 `.env`에는 `API_BASE_URL` 중심의 공개 가능한 설정만 있는가?
- `SUPABASE_SERVICE_ROLE_KEY`가 프론트엔드에 없는가?
- `GEMINI_API_KEY`, `OPENAI_API_KEY`, `UPSTASH_REDIS_REST_TOKEN`을 프론트엔드에 두지 않았는가?
- 오류 메시지나 로그에 token이 그대로 출력되지 않는가?

## 제출 문서

- 실행 명령이 적혀 있는가?
- Swagger 또는 Streamlit 실행 결과가 정리되어 있는가?
- 오류와 해결 기록이 있는가?
- 선택 기능과 필수 기능이 구분되어 있는가?
