## 단계별 프롬프트 계획

| 단계 | 목표 | Codex에게 요청할 내용 |
| --- | --- | --- |
| 1 | 화면 흐름 정리 | 아직 코드 작성 없이 사용자 흐름만 정리 |
| 2 | mock UI 작성 | 백엔드 없이 화면 먼저 구현 |
| 3 | 상태 관리 추가 | `st.session_state`로 입력/결과 유지 |
| 4 | API 연결 | `API_BASE_URL`, timeout, status code 처리 |
| 5 | 오류/로딩 상태 | 실패와 대기 상태 표시 |
| 6 | UI 리뷰 | 화면 흐름, 보안, 코드 구조 점검 |
