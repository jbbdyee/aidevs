# 01. Screen Design

이 문서는 최종 프로젝트 화면을 설계할 때 사용하는 간단한 템플릿입니다.

## 프로젝트 화면 목표

```text
개인화 AI 챗봇 서비스 통합 구현
```

## 화면 구성

| 영역 | 포함할 내용 |
| --- | --- |
| 사이드바 | API 주소, 회원가입, 로그인, 로그아웃, 현재 사용자 |
| 챗봇 탭 | 사용자 질문, assistant 응답, 응답 생성 중 상태 |
| 대화 기록 탭 | 사용자별 대화 기록 조회 |
| 서비스 로그 탭 | 회원가입, 로그인, 챗봇 호출 로그 조회 |
| 배포 점검 탭 | Render, Upstash, Streamlit Community Cloud 점검 |

기본 실습은 `backend_mock`과 연결합니다. 실제 배포 흐름을 확인할 때는 `backend_service`로 바꿔 연결합니다.

## 화면별 필수 상태

| 화면 | 확인할 상태 |
| --- | --- |
| 사이드바 | 로그인 전, 로그인 성공, 로그아웃 후 |
| 챗봇 탭 | 빈 대화, 사용자 질문 입력, 응답 생성 중, 응답 표시, 오류 |
| 대화 기록 탭 | 기록 없음, 기록 있음, API 오류 |
| 서비스 로그 탭 | 로그 없음, 로그 있음, API 오류 |
| 배포 점검 탭 | local mock, service backend, deploy 준비 상태 |

## UX 상태

- [ ] 로그인 전 상태
- [ ] 로그인 성공 상태
- [ ] 백엔드 연결 실패 상태
- [ ] AI 응답 생성 중 상태
- [ ] 대화 기록이 없는 상태
- [ ] 서비스 로그가 없는 상태
