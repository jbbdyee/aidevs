# 02. 화면 설계서 샘플

프로젝트명: AI 서비스 로그 분석 및 운영 대시보드 구축

## 화면 목록

| 화면 | 목적 | 주요 컴포넌트 |
| --- | --- | --- |
| 메인 대시보드 | 전체 상태와 주요 지표 확인 | metric, chart, realtime log area |
| 로그 입력 | 테스트용 서비스 로그 생성 | form, selectbox, text_area, button |
| 로그 상세 | 단일 로그의 상세 정보 확인 | table, json viewer |
| 피드백 입력 | AI 답변 품질 개선 피드백 기록 | rating, text_area, button |
| 설정/상태 | API_BASE_URL, backend 상태 확인 | text, button, status message |
| 에러 화면 | API 연결 실패 또는 입력 오류 안내 | warning, error, retry button |

## 메인 대시보드 와이어프레임

```text
┌────────────────────────────────────────────┐
│ AI 서비스 실시간 로그 대시보드              │
├───────────────┬───────────────┬────────────┤
│ 전체 로그 수   │ 에러 로그 수    │ 평균 응답시간 │
├────────────────────────────────────────────┤
│ 실시간 로그 스트림                          │
│ - 13:01 info chat-api AI 응답 완료          │
│ - 13:02 error llm-api Gemini 호출 실패      │
├────────────────────────────┬───────────────┤
│ 최근 로그 테이블             │ level별 차트    │
└────────────────────────────┴───────────────┘
```

## 디자인 기준

| 항목 | 기준 |
| --- | --- |
| 버튼 | 주요 동작은 명확한 텍스트 버튼으로 표시 |
| 입력폼 | 로그 level, source, message를 한 영역에 배치 |
| 테이블 | 최근 로그는 최신순으로 표시 |
| 차트 | level별 개수, 상태코드 분포를 간단히 표시 |
| 색상 | error/warning/info 상태를 구분하되 과도한 색 사용은 피함 |
| 여백 | 지표, 로그, 차트 영역을 구분해 읽기 쉽게 배치 |

## 사용자 액션과 시스템 반응

| 사용자 액션 | 시스템 반응 |
| --- | --- |
| 로그 생성 버튼 클릭 | POST `/logs` 호출 후 성공/실패 메시지 표시 |
| 최근 로그 새로고침 | GET `/logs` 호출 후 테이블 갱신 |
| 실시간 보기 클릭 | GET `/stream/logs` 연결 후 새 이벤트 표시 |
| 피드백 저장 | POST `/feedback` 호출 후 저장 결과 표시 |
| 백엔드 상태 확인 | GET `/health` 호출 후 JSON 표시 |
| API 오류 발생 | `st.error`로 원인 표시 |

## 체크리스트

- [ ] 모든 화면의 와이어프레임이 누락 없이 작성되었다.
- [ ] 정보 계층구조가 논리적이다.
- [ ] 버튼, 입력폼, 테이블, 차트 스타일이 일관된다.
- [ ] 클릭, 호버, 스크롤, 필터, 새로고침 같은 사용자 액션의 시스템 반응이 정의되었다.
- [ ] API 호출 중 로딩 상태와 실패 상태가 정의되었다.
