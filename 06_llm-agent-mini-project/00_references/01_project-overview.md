# 01 Project Overview

## 주제

복합 API 연계 일정 조정 에이전트

## 문제 상황

여러 사람이 참여하는 회의나 상담 일정을 잡을 때는 참석자, 날짜 범위, 회의 길이, 선호 시간, 이미 잡힌 일정 같은 정보를 함께 확인해야 합니다.

일정 조정 에이전트는 사용자의 요청을 읽고, 필요한 정보를 파악하고, 일정 데이터 조회 Tool을 호출한 뒤 가능한 시간을 제안합니다. 정보가 부족하거나 일정이 충돌하면 바로 답을 꾸며내지 않고 추가 질문, 대체 제안, fallback 응답을 만들어야 합니다.

## 필수 흐름

```text
사용자 요청 입력
-> 요청 분석
-> 필요한 Tool 결정
-> Tool 호출
-> Tool 결과 검증
-> 오류 또는 정보 부족 판단
-> 재시도, 추가 질문, fallback, 최종 응답
```

## 필수 구현 기준

| 항목 | 기준 |
| --- | --- |
| Agent 구조 | LangGraph StateGraph를 사용합니다. |
| Tool | 2개 이상의 Tool을 구현합니다. |
| 상태 관리 | 요청, 메시지, Tool 결과, 오류 횟수, 반복 횟수, 최종 답변을 State에 저장합니다. |
| 오류 처리 | Tool 선택 오류, 파라미터 누락, 데이터 부족, 응답 불일치를 감지합니다. |
| 자기 성찰 | 오류 원인을 분석하고 재시도 또는 fallback 전략을 선택합니다. |
| 문서화 | 에이전트 아키텍처 설계서와 시험 결과 보고서를 작성합니다. |

## 05 과정과의 연결

05 과정에서 배운 Prompt, Structured Output, Function Calling, Tool Use, RAG/Memory, LangGraph를 프로젝트 안에서 필요한 만큼 적용합니다.

## 07 과정과의 연결

06 과정은 단일 에이전트 프로젝트입니다. 07 과정에서는 멀티 에이전트 협업, Docker Compose, 배포 자동화, 장애 복구, 운영 모니터링으로 확장합니다.
