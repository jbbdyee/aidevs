# 04 Self Reflection Feedback Loop

Self-Reflection은 Agent가 자신의 결과를 검토하고, 오류가 있으면 수정 전략을 선택하는 흐름입니다.

## 감지할 오류 유형

| 오류 유형 | 예시 | 감지 기준 |
| --- | --- | --- |
| 할루시네이션 | Tool 결과에 없는 시간을 제안함 | 최종 답변과 `tool_results` 비교 |
| Tool 선택 오류 | 일정 조회 없이 답변을 생성함 | `required_tools`와 `tools_called` 비교 |
| 파라미터 누락 | 참석자나 날짜 없이 Tool 호출 | Tool 입력값 검증 |
| 응답 불일치 | 불가능한 시간을 가능하다고 말함 | 후보 시간 목록과 최종 답변 비교 |
| 데이터 부족 | 가능한 시간이 없음 | `available_slots`가 비어 있음 |

## 피드백 루프

```text
오류 감지
-> 원인 분석
-> 수정 전략 선택
-> 재실행 또는 fallback
-> 결과 검증
-> 최종 응답
```

## 수정 전략 예시

| 상황 | 전략 |
| --- | --- |
| 파라미터 누락 | 사용자에게 추가 질문 |
| Tool 선택 오류 | `decide_tool` 노드로 되돌아감 |
| Tool 일시 실패 | 1회 재시도 후 fallback |
| 일정 충돌 | 대체 날짜 또는 회의 시간 축소 제안 |
| 응답 불일치 | Tool 결과만 사용하도록 응답 재생성 |

## 보고서에 기록할 항목

- 오류 유형
- 감지 기준
- 재시도 횟수
- fallback 조건
- 수정 전후 결과
- 프롬프트 또는 파라미터 변경 이력

Self-Reflection은 복잡하게 만들 필요가 없습니다. 초보자 프로젝트에서는 `if` 조건과 간단한 검증 함수만으로도 충분히 시작할 수 있습니다.
