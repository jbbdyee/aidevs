# Lab 04. CI/CD Build Check

## 목표

GitHub Actions에서 Docker build를 자동 확인하는 흐름을 이해합니다.

## 실습

1. `.github/workflows/docker-build-check.yml` 파일을 확인합니다.
2. 어떤 폴더에서 Docker build를 실행하는지 확인합니다.
3. 실제 프로젝트 저장소에 적용할 때 경로를 어떻게 바꿔야 하는지 정리합니다.

## 확인 질문

- GitHub Actions에서 API Key를 코드에 쓰면 안 되는 이유는 무엇인가?
- Docker build 실패가 배포 전에 발견되면 어떤 장점이 있는가?
